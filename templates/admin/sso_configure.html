Setup Guide
{% if provider_type == 'google_oauth' %}
Google Cloud Console Setup:
  1. Go to Google Cloud Console
  2. Create/select a project
  3. Go to "APIs & Services" → "Credentials"
  4. Create OAuth 2.0 Client ID
  5. Set redirect URI: {{ request.url_root }}auth/sso/callback/oauth
  6. Copy Client ID and Client Secret here
{% elif provider_type == 'azure_ad' %}
Azure AD Setup:
  1. Go to Azure Portal
  2. Navigate to "Azure Active Directory"
  3. Go to "App registrations" → "New registration"
  4. Set redirect URI: {{ request.url_root }}auth/sso/callback/azure_ad
  5. Generate client secret
  6. Copy Tenant ID, Client ID, and Client Secret
{% elif provider_type == 'saml' %}
SAML Setup:
  1. Configure your SAML Identity Provider
  2. Set ACS URL: {{ request.url_root }}auth/sso/callback/saml
  3. Set Entity ID: {{ request.url_root }}sso/metadata
  4. Download IdP certificate
  5. Configure attribute mappings for email
{% else %}
OAuth 2.0 Setup:
  1. Register application with your OAuth provider
  2. Set redirect URI: {{ request.url_root }}auth/sso/callback/oauth
  3. Obtain client credentials
  4. Configure endpoint URLs
  5. Test the configuration
{% endif %}
Testing:

After saving, use the "Test" button on the dashboard to verify connectivity.

Security Notes:
  • Client secrets are encrypted in database
  • Use HTTPS in production
  • Review provider permissions
  • Monitor authentication logs
